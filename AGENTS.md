# 项目规则

## 验证行为规则（重要）

执行任何任务时，遵守以下规则：

1. **只验证一次**：构建、重启、修改完成后，只执行一次验证命令（如 `pgrep`、`grep`、`curl`），确认成功后立即回复用户。
2. **不要重复检查**：验证通过后，禁止再次执行相同的验证命令。即使不确定结果，也不要重复。
3. **完成即停**：任务完成后，直接回复"已完成"或简要总结，不要追加额外的检查命令。
4. **识别死循环**：如果发现自己连续执行了两次以上相同的命令，立刻停止并回复当前状态。

## 示例

**正确做法：**
```
wails build → 成功 → open app → pgrep 确认运行 → 回复"已启动"
```

**错误做法：**
```
wails build → 成功 → open app → pgrep → pgrep → pgrep → pgrep → 卡住
```
