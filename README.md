# PDF Knowledge Base

一个免安装的文件标签管理工具，通过知识网络图谱可视化文档之间的关联关系。支持 PDF、Office 文档、图片、视频等多种文件类型。

---

## 功能概览

### 📂 文件扫描与管理
- **递归扫描文件夹**：`filepath.WalkDir` 高效遍历，支持 pdf / docx / xlsx / pptx / 21 种图片格式 / 8 种视频格式
- **单事务批量写入**：扫描结果一次性写入 SQLite，3000 文件 <10ms
- **智能重定位**：文件改名/移动后，通过「文件大小 + 修改时间」指纹自动找回并保留标签
- **选择路径浏览**：选择文件夹后递归展示该目录及子目录下所有文件
- **文件类型筛选**：左侧栏按类型分组筛选，实时显示各类型数量
- **未分类筛选**：一键查看无标签的文件

### 🏷️ 标签管理
- **手动打标签**：输入新标签名或从已有标签列表点击添加
- **自动补全**：输入时模糊匹配已有标签
- **批量打标签**：多选文件后一次性添加同一标签
- **批量移除标签**：多选文件后批量移除指定标签
- **重命名 / 删除标签**：支持全局重命名和删除（级联清理关联）
- **矩阵标签选择器**：弹出式标签选择面板，虚线边框 `# 前缀` 样式

### 📋 批量操作
- **复选框多选**：单击勾选 + Shift 范围选 + Ctrl 逐选
- **全选 / 取消选择**
- **批量移除文件**：选中后从列表移除（不删真实文件）
- **批量打标签 / 批量移除标签**

### 🔍 搜索与筛选
- **全文搜索**：按文件名、路径、标签名模糊匹配
- **标签筛选**：点击标签筛选文档，支持多标签叠加
- **文件类型筛选**：按类型分组点击筛选
- **文件夹模式**：选择路径后只显示该文件夹及其子目录下的文件

### 🕸️ 知识网络图谱
- **PDF 关联网络**：共享标签的文档之间连线，权重 = 共享标签数
- **标签关联网络**：共现在同一文档的标签之间连线，权重 = 共现次数
- **力导向布局**：vis.js 渲染，节点大小映射关联度，边粗细映射权重
- **交互操作**：拖拽、缩放、双击跳转文件详情、搜索高亮聚焦

### 🖥️ 系统集成
- **打开文件**：用系统默认程序打开选中文件
- **打开目录**：打开文件所在目录并选中文件
- **Windows 右键菜单**：`--install-shell` 安装右键「添加标签(T)」菜单项（无需管理员权限），`--uninstall-shell` 卸载
- **标签选择器模式**：右键文件弹出独立小窗口快速打标签

### ⚙️ 设置
- **文件类型配置**：可选启用/禁用的文件类型（pdf、docx、image、video 等）
- **设置持久化**：保存在 `~/.pdf-knowledge-base/settings.json`

| 组件 | 技术 |

## 开发环境搭建

### 前提条件

1. **Go 1.21+**：https://go.dev/dl/
2. **Wails CLI**：
   ```bash
   go install github.com/wailsapp/wails/v2/cmd/wails@latest
   ```
3. **CGO 支持**（SQLite 需要）：
   - Windows：安装 TDM-GCC 或 MinGW-w64
   - macOS：`xcode-select --install`
   - Linux：`sudo apt install gcc`

### 安装依赖

```bash
go mod tidy
```

### 开发模式

```bash
wails dev
```

### 构建生产版本

```bash
wails build
```

构建完成后，可执行文件在 `build/bin/` 目录下。

## 数据存储

数据库文件存储在用户目录下：

| 系统 | 路径 |
|------|------|
| Windows | `C:\Users\<用户名>\.pdf-knowledge-base\data.db` |
| macOS | `~/.pdf-knowledge-base/data.db` |
| Linux | `~/.pdf-knowledge-base/data.db` |

文件不会被移动或修改，软件只记录路径和标签信息。

## 性能基准测试

使用 `go test -bench=. ./internal/scanner/` 测试扫描模块：

| 文件数量 | 扫描耗时 |
|----------|----------|
| 100 文件 | ~1.1ms |
| 1000 文件 | ~1.8ms |
| 3000 文件 | ~5.7ms |

完整扫描流程（遍历 + DB 写入 + 智能匹配）：<70ms（3000 文件规模）。

## 已知问题与待改进

### 功能层面
- [ ] **增量扫描**：当前每次全量扫描，文件量大时较慢；计划实现基于 mtime 的增量扫描
- [ ] **PDF 内容提取**：已预留 `Extractor` 接口，当前为空实现；计划集成 PDF 文本提取
- [ ] **全文搜索**：依赖内容提取，实现后可搜索 PDF/Word 文档内容
- [ ] **标签自动推荐**：从文档内容自动推荐关键词标签
- [ ] **文件夹计数实时更新**：选择路径后文件夹数量刷新时机需优化

### 技术层面
- [ ] **vis.js 离线打包**：生产环境需将 CDN 资源改为本地文件
- [ ] **跨平台右键菜单**：当前仅支持 Windows 注册表方式，macOS/Linux 需要适配
- [ ] **大文件量性能**：10000+ 文件场景下前端渲染可能卡顿，需虚拟列表优化
- [ ] **数据库迁移**：升级时的 schema 迁移机制需要更健壮

### UI/UX
- [ ] 批量操作按钮布局待调整优化
- [ ] 文件夹模式下左侧计数与筛选逻辑的一致性
- [ ] 图谱视图节点过多时的性能优化
